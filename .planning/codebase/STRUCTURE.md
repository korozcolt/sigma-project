# Codebase Structure

**Analysis Date:** 2026-03-24

## Directory Layout

```
sigma-project/
├── app/               # Application core logic (Models, Providers, Filament)
├── bootstrap/         # Framework booting and cache
├── config/            # Application configuration files
├── database/          # Migrations, Factories, Seeders
├── docs/              # Project documentation (Docusaurus)
├── lang/              # Translation files
├── public/            # Publicly accessible files and index.php
├── resources/         # Views (Blade/Volt), raw CSS/JS assets
├── routes/            # Web, API, and Console route definitions
├── storage/           # Logs, compiled views, file uploads
└── tests/             # Pest unit and feature tests
```

## Directory Purposes

**app/**
- Purpose: Core application code
- Contains: PHP classes (`.php`)
- Key files: `Models/User.php`
- Subdirectories: `Http/`, `Models/`, `Providers/`, `Filament/` (likely if Filament is used)

**config/**
- Purpose: All configuration files
- Contains: PHP array returning files
- Key files: `app.php`, `database.php`

**database/**
- Purpose: Database schema and seeding
- Contains: Migrations, seeders, factories
- Subdirectories: `migrations/`, `seeders/`, `factories/`

**resources/**
- Purpose: Frontend assets and views
- Contains: Blade templates, CSS (Tailwind), JS
- Key files: `css/app.css`, `js/app.js`
- Subdirectories: `views/` (standard Blade), `views/livewire/` (Volt components)

**routes/**
- Purpose: URL to logic mapping
- Contains: Route definition files
- Key files: `web.php`, `console.php`

**tests/**
- Purpose: Automated Pest testing
- Contains: Test classes
- Subdirectories: `Feature/`, `Unit/`

**docs/**
- Purpose: Documentation site
- Contains: Markdown files, likely parsed by Docusaurus (as `docusaurus.json` exists)

## Key File Locations

**Entry Points:**
- `public/index.php`: Web application entry point
- `artisan`: CLI entry point

**Configuration:**
- `.env`: Environment variables
- `composer.json`: PHP dependencies and autoloading
- `package.json`: Node dependencies and scripts
- `vite.config.js`: Asset compilation configuration

**Core Logic:**
- `app/Models/`: Database interactions

**Testing:**
- `phpunit.xml`: Test runner configuration
- `tests/`: Pest test files

## Naming Conventions

**Files:**
- `PascalCase.php` for PHP classes (Models, Controllers)
- `kebab-case.blade.php` for Blade templates
- `kebab-case.php` for configuration files

**Directories:**
- PascalCase for `app/` directories matching namespaces (e.g. `Http`, `Models`)
- lowercase for root directories (`resources`, `database`)

**Special Patterns:**
- `*.blade.php` for view files

## Where to Add New Code

**New Frontend Page/Component:**
- Implementation: `resources/views/livewire/[name].blade.php` (if using Volt)
- Route: add to `routes/web.php`

**New Admin Panel Feature:**
- Implementation: `app/Filament/Resources/`

**New Database Entity:**
- Model: `app/Models/`
- Migration: `database/migrations/`
- Factory: `database/factories/`

## Special Directories

**node_modules/**
- Purpose: npm dependencies
- Source: Generated by `npm install`
- Committed: No

**vendor/**
- Purpose: Composer dependencies
- Source: Generated by `composer install`
- Committed: No
